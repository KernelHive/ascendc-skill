# 接口简介

## 概述

为加速模型算力释放，CANN（Compute Architecture for Neural Networks）提供了一系列丰富且深度优化、硬件亲和的高性能算子，如Neural Network、Digital Vision Pre-Processing算子等，为神经网络在昇腾硬件上加速计算奠定了基础。

为方便开发者调用算子，提供了一套基于C的API执行算子（如aclnn或acldvpp API等），无需提供IR（Intermediate Representation）定义，方便开发者高效构建模型与应用开发，算子API的调用流程如图1-1所示。

此外，本文档还提供了不同深度学习框架IR（Intermediate Representation）定义的算子规格信息，支撑开发者自行构建网络模型。

**说明**

- 本文档提供的CANN算子均为系统支持的算子，对于系统不支持的算子，支持开发者自定义，自定义算子开发指导请参见《Ascend C算子开发指南》。
- 注意aclnn表示以aclnn为前缀的算子API，如NN算子接口和融合算子接口；acldvpp表示以acldvpp为前缀的API，包括DVPP算子接口。

## 接口说明

本文档提供了算子规格说明、算子API等，具体如表1-1所示，产品支持情况参见表1-2。

文档重点阐述了算子定义、功能描述、参数说明、约束限制和调用示例等，指导开发者快速上手调用算子。

如需调用算子API，注意需要include依赖的文件，一般头文件默认在“${INSTALL_DIR}/include/”目录，库文件默认在“${INSTALL_DIR}/lib64/”目录，${INSTALL_DIR}请替换为CANN软件安装后文件存储路径。若安装的Ascend-cann-toolkit软件包，以root安装举例，则安装后文件存储路径为：`/usr/local/Ascend/ascend-toolkit/latest`。

**说明**

- 对于各类算子接口或算子规格/清单中未声明支持的场景（如产品型号、数据类型、数据格式、数据维度等），不推荐开发者使用，当前版本不保证算子调用效果。
- 调用算子过程中可能会遇到各类异常，请参考《故障处理》中“故障案例集 > 算子执行问题”，文档列举了典型/高频的算子执行问题，辅助开发者进行问题定位和解决。

| 分类 | 简介 | 依赖的头文件或库文件 | 备注 |
|------|------|----------------------|------|
| 公共接口 | 调用NN算子、融合算子、DVPP算子API过程中依赖的公共Meta接口，如创建aclTensor、aclScalar、aclIntArray等。 | - 头文件：aclnn/acl_meta.h<br>- 库文件：libnnopbase.so | - |
| NN算子接口 | Neural Network算子，CANN内置的基础算子，接口前缀为aclnnXxx，主要覆盖TensorFlow、PyTorch、MindSpore、ONNX等框架中深度学习算法相关的计算类型，例如常见的Softmax、MatMul、Convolution等。目前该类算子API在整个算子库中占最大比重。 | 除aclnnInit和aclnnFinalize外，其余接口依赖文件如下：<br>- 头文件：aclnnop/aclnn_*.h<br>- 动态库：libopapi.so（可被划分如下领域，按需选装）：<br>  – libaclnn_ops_infer.so（网络算子推理库）<br>  – libaclnn_ops_train.so（网络算子训练库）<br>  – libaclnn_math.so（数学算子库）<br>  – libaclnn_rand.so（随机数算子库）<br>- 静态库：<br>  – libaclnn_ops_infer_static.a（网络算子推理库）<br>  – libaclnn_ops_train_static.a（网络算子训练库）<br>  – libaclnn_math_static.a（数学算子库）<br>  – libaclnn_rand_static.a（随机数算子库） | 本质是一套基于C语言的API，可直接调用API执行算子，无需提供额外的IR定义。该调用方式被称为单算子API执行调用，详细介绍参见《应用开发指南 (C&C++)》中“单算子调用 > 单算子调用流程”。<br><br>调用NN、融合算子接口时，会直接调用算子二进制包（Ascend-cann-kernels）中已经编译好的算子，无需再编译算子。算子二进制包和库文件介绍请参见《CANN 软件安装指南》。<br><br>为兼容旧版本，旧版本NN/融合算子头文件路径是aclnnop/level2/aclnn_*.h，但后续版本该路径会废弃，建议使用新路径aclnnop/aclnn_*.h，防止后续版本出现兼容性问题。 |
| 融合算子接口 | CANN内置的融合算子，接口前缀为aclnnXxx，由多个独立的基础“小算子”（如向量Vector、矩阵Cube等）融合而成，大算子功能与多个小算子功能等价，且大算子性能收益往往优于小算子。例如常见的Flash Attention、通算融合算子（简称MC2算子）等。 | - aclnnInit和aclnnFinalize依赖头文件为aclnn/aclnn_base.h、库文件为libnnopbase.so。 | 除了本文档提供的融合算子，您还可以单击Link访问Gitee cann-ops-adv仓获取代码开放的融合算子。<br><br>当前融合算子不支持使用昇腾虚拟化实例。 |
| DVPP算子接口 | Digital Vision Pre-Processing算子，接口前缀为acldvppXxx，提供高性能视频/图片编解码、图像裁剪缩放等预处理API。 | - 头文件：acldvppop/acldvpp_base.h和acldvppop/acldvpp_op_api.h<br>- 库文件：libacl_dvpp_op.so | 调用DVPP算子接口时，无需再编译算子。 |
| Ascend IR算子规格说明 | 罗列了基于Ascend IR定义的算子信息。 | - | - |
| AI框架算子支持清单 | 罗列了主流AI框架原生IR定义的算子信息（如TensorFlow、Caffe等）。 | - | - |

## 公共接口返回码

调用公共接口时，常见的返回码如下表所示。

| 状态码名称 | 状态码 | 状态码说明 |
|------------|--------|------------|
| ACLNN_SUCCESS | 0 | 成功。 |
| ACLNN_ERR_PARAM_NULLPTR | 161001 | 参数校验错误，参数中存在非法的 nullptr。 |
| ACLNN_ERR_PARAM_INVALID | 161002 | 参数校验错误，如输入的两个数据类型不满足输入类型推导关系。 |
| ACLNN_ERR_RUNTIME_ERROR | 361001 | API 内存调用 npu runtime 的接口异常。 |
| ACLNN_ERR_INNER_XXX | 561xxx | API 发生内部异常。 |

详细的错误消息，可通过《应用开发指南 (C&C++)》中“acl API 参考 > 系统配置 > aclGetRecentErrMsg”接口获取。

## 说明

常见的内部异常场景如下，其余异常请根据报错排查问题或联系技术支持（您可以获取日志后单击 Link 联系技术支持）。

- **561101（ACLNN_ERR_INNER_CREATE_EXECUTOR）**：API 内部创建 aclOpExecutor 失败（可能因为操作系统异常）。
- **561102（ACLNN_ERR_INNER_NOT_TRANS_EXECUTOR）**：API 内部未调用 uniqueExecutor ReleaseTo。
- **561103（ACLNN_ERR_INNER_NULLPTR）**：aclnn API 内部发生异常，出现了 nullptr 的异常。

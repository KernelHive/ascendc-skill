## 概述

图模式是神经网络模型的一种运行模式。在图模式下，用户首先将模型的计算过程构造为一张图，然后通过图引擎（GE）将图下发到昇腾硬件执行。相对于逐个算子依次下发的方式，图模式下 GE 可以通过以下技术手段提升性能：

- 计算图优化
- 多流并行
- 内存复用
- 模型下沉

这些手段能够加速模型执行效率，并减少模型内存占用。

算子入图的开发流程如下：

1. 算子工程创建完成后，基于工程代码框架完成算子原型定义、kernel 侧算子实现、host 侧 tiling 实现，并完成算子入图开发。
2. 通过工程编译脚本完成算子的编译部署。
3. 之后即可基于图 IR 执行算子，例如单算子模型执行或通过 IR 构图的方式调用自定义算子。

该开发流程以“6.7 工程化算子开发”为基础，除了需要提供工程化算子开发中的算子实现文件外，还需额外交付算子入图的代码文件。

## 步骤1：环境准备

1. CANN 软件安装请参考“2 环境准备”。
2. 创建算子工程：使用 `msOpGen` 工具创建算子开发工程。

## 步骤2：算子实现

- **算子原型定义**：通过原型定义描述算子输入输出、属性等信息，以及算子在 AI 处理器上的相关实现信息，并关联 tiling 实现等函数。
- **Kernel 侧算子实现和 Host 侧 tiling 实现**：请参考“6 算子实现”。工程化算子开发支持开发者调用 Tiling API，基于 CANN 提供的编程框架进行 tiling 开发。Kernel 侧也提供对应接口，方便开发者获取 tiling 参数。具体内容请参考：
  - 6.7.4 Kernel 侧算子实现
  - 6.7.5 Host 侧 Tiling 实现
  相关额外约束也在上述章节中说明。

## 步骤3：算子入图（GE 图）开发

在算子入图场景下，需要提供 shape 推导等算子入图适配函数的实现。

## 步骤4：编译部署

通过工程编译脚本完成算子的编译部署，分为两种方式：

- 算子包编译
- 算子动态库编译

## 步骤5：图编译和图执行

基于图 IR 执行算子，例如：

- 单算子模型执行
- IR 构图方式调用自定义算子

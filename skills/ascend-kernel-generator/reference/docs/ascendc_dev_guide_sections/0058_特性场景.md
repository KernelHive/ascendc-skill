#### 特性场景

为避免执行通信任务的环境中硬件闪断导致通信中断，通算融合算子可通过配置编译宏与环境变量，开启重执行能力。开启重执行后，AI CPU 在检测到环境异常时，通过下图所示机制通知 AI Core 重新下发通信任务，避免由于硬件闪断造成的通信中断，从而提升通信稳定性。

当前支持情况如下：

- Atlas A2 训练系列产品/Atlas A2 推理系列产品：不支持通算融合算子的重执行
- Atlas A3 训练系列产品/Atlas A3 推理系列产品：支持通算融合算子的重执行

## 开启重执行的条件

开启重执行需满足以下条件：

1. **内存地址要求**  
   通算融合算子的输出内存地址和输入内存地址不相同。

2. **配置编译宏**  
   算子编译时，需配置编译宏 `AICORE_EXCEPTION_RESTART`，示例如下：  
   ```cpp
   add_ops_compile_options(ALL OPTIONS -DAICORE_EXCEPTION_RESTART)
   ```  
   具体的编译宏配置阶段和方式请参考《支持自定义编译选项》。

3. **配置环境变量**  
   配置 HCCL 重执行环境变量 `HCCL_OP_RETRY_ENABLE`，开启重执行的检测和上报能力。该环境变量的说明请参考《环境变量参考》中“集合通信 > HCCL_OP_RETRY_ENABLE”部分。  
   请在算子执行前设置该环境变量，具体配置如下：  
   ```bash
   # server 内 L0 和 server 间 L1 均需配置为 1，不支持跨超节点，L2 配置为 0
   export HCCL_OP_RETRY_ENABLE="L0:1, L1:1, L2:0"
   ```

> **注意**  
> 开启重执行后，若 AI Core 第一次下发通信任务后通信中断，默认只重执行一次。如需修改重执行次数或重传间隔时间，请参考《环境变量参考》中“集合通信 > HCCL_OP_RETRY_PARAMS”部分。
